# Goal File Format

Save goal files to `{kb_path}/goals/[area].md` using this format. Replace bracketed placeholders with actual content from the goal creation conversation.

## Growth Goal Template

```
# [Life Area Name]

## Needle
[What moves. How it's measured. What weekly movement looks like.]

## Why This Matters
[In the user's own words — why this life area matters to them. This gets referenced by accountability, so capture the real motivation, not a polished version.]

## Commitments

### [Commitment Name]
- **What:** [Specific action, specific time, specific frequency]
- **External structure:** [Who notices if this doesn't happen. What's the cost of skipping.]
- **Counts:** [What actions meet this commitment]
- **Does not count:** [What actions feel like meeting it but don't — be specific, close loopholes]

### [Additional Commitment]
- **What:** [...]
- **External structure:** [...]
- **Counts:** [...]
- **Does not count:** [...]

## When-Thens
- When [specific trigger], I will [specific response]
- When [specific trigger], I will [specific response]

## WOOP
- **Wish:** [The goal]
- **Outcome:** [Vivid image of achieving it]
- **Obstacle:** [The internal barrier — not external. The pattern, feeling, or belief that gets in the way.]
- **Plan:** [Implementation intention specifically for the obstacle]

## Current Status
[Updated by the system during daily logs and weekly reflections. Contains last needle check date, recent commitment data, and any pattern notes.]
```

## Boundary Goal Template

For goals where the aim is to bound or decrease something (used when balance_challenge = "balancing" for the expanding area). Save to the same location: `{kb_path}/goals/[area].md`.

```
# [Area Name] — Boundary Goal

## Ceiling
[What's the upper bound. How violation is measured. What "staying under" looks like weekly.]

## Why This Matters
[What is this boundary protecting? Name the other life areas that suffer when this area is unbounded. In the user's own words.]

## STOP Commitments

### [STOP Commitment Name]
- **What to stop/limit:** [Specific behavior to stop, specific boundary, specific trigger for stopping]
- **External structure for enforcement:** [Who enforces this. What mechanism prevents override. NOT willpower.]
- **Violation:** [What actions break this boundary — be specific]
- **Not a violation:** [What might feel like breaking it but is actually fine]

### [Additional STOP Commitment]
- **What to stop/limit:** [...]
- **External structure for enforcement:** [...]
- **Violation:** [...]
- **Not a violation:** [...]

## When-Thens
- When [high-pull moment], I will [specific response that holds the line]
- When [rationalization trigger], I will [specific response]

## WOOP
- **Wish:** [The bounded state — life with the boundary held]
- **Outcome:** [Vivid image of the bounded life — what flourishes when this boundary is kept]
- **Obstacle:** [The PULL — the internal drive or belief that makes overriding feel justified. Not external barriers.]
- **Plan:** [Implementation intention specifically for the pull]

## Current Status
[Updated by the system during daily logs and weekly reflections.]
```

## Format Rules

- Use the user's actual words wherever possible, especially in "Why This Matters" and "WOOP Obstacle"
- Keep commitment definitions tight — ambiguity is where rationalizations hide
- The "Does not count" / "Not a violation" list should anticipate the specific ways the user might claim partial credit
- Current Status section starts empty and gets updated over time
