# Weekly Reflection File Format

Save weekly reflections to `{kb_path}/log/weekly/YYYY-MM-DD.md` using this format. Write each section as it's completed during the reflection conversation — do not wait until the end.

## Template

```
# Weekly Reflection — YYYY-MM-DD

## Needle Review

| Life Area    | Moved? | Data |
|-------------|--------|------|
| [Area]      | Yes/No | [Specific measurement or observation] |

## Commitment Scorecard

| Commitment          | Mon | Tue | Wed | Thu | Fri | Sat | Sun | Hit Rate |
|---------------------|-----|-----|-----|-----|-----|-----|-----|----------|
| [Commitment name]   |  Y  |  N  |  -  |  Y  |  N  |  -  |  -  | 2/4      |

## Patterns

- [Pattern observed, with data]
- [Pattern observed, with data]

## Harsh Truth

**Question asked:** [The question]

**Response:** [What the user said — in their words]

**Outcome:** [Did they change course, make an eyes-open choice, or rationalize?]

## Next Week's Commitments

- [Commitment — same as last week / modified / new]
- [Any new external structure added]

## WOOP — [Life Area]

- **Wish:** [Goal]
- **Outcome:** [Vivid image]
- **Obstacle:** [Internal barrier — the user's actual words]
- **Plan:** [Implementation intention]

## Notes

[Anything else significant from the reflection conversation]
```

## Format Rules

- Use the user's actual words in the Harsh Truth response and WOOP obstacle
- Hit rates in the scorecard help spot trends across weeks
- Notes section captures anything that doesn't fit the structured sections
- Each section is appended as it's completed — the file builds during the conversation
- Sections not included at the user's reflection depth level should be omitted entirely (don't add empty sections)
